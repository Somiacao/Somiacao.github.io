Apache Spark

Spark是一个统一的分布式内存计算引擎，包括一组用于在计算机集群上进行并行数据处理的函数。

可运行在笔记本电脑上，也可运行于数千台服务器的集群上

spark是由许多不同的组件组成的

spark的关键部分

统一

spark的关键驱动目标是为编写大数据应用程序提供一个统一的平台。spark的设计目的是支持广泛的数据分析任务，
从简单的加载和SQL查询到机器学习和流计算，这些都通过相同的计算引擎和一致的api集合实现。

spark提供了一致的、可组合的api,使用这些api从较小的部分或现有的库中构建应用程序。spark的API也被设计为通过
优化在用户程序中组合的不同库和函数来实现高性能。

计算引擎

spark处理从存储系统加载数据并在其上执行，但最终数据并不永久存储在spark中。spark可和多种存储系统结合使用，如
kafka、HBase、Hive、HDFS以及关系型数据库。

Hadoop和spark的区别：
Hadoop包括一个存储系统（Hadoop文件系统HDFS，用低成本的商品服务器集群做存储）和一个紧密集成的计算引擎
（MapReduce）。存储系统和计算引擎无法割裂。

函数库

spark最终的组件是它的库，它以统一引擎的设计为基础，为公共数据分析任务提供统一的API。

spark主要的库：SQL和结构化数据的库（spark SQL）、机器学习（MLlib）、流处理（Spark流和新的结构化流处理
structured Streaming）和图形分析（GraphX）。

spark产生的背景

由于散热方面的限制，硬件开发人员停止让单个处理器运行的更快，转而使用相同的速度增加更多的并行CPU内核。需要
修改程序以增加并行性，以便更快的运行，这为新的编程模式（spark）创造了舞台。

搜集了大量数据，处理这些数据需要并行度高的计算引擎，需要运行在集群之上。

过去的50年中开发的软件不能自动伸缩，传统的处理数据程序的编程模式不能满足新的编程需求。

spark的基本框架

一组机器本身并不强大，需要一个框架来协调它们之间的工作。spark就是这样做的，管理和协调跨集群计算机上的数据执行
任务。spark将用于执行任务的集群交由集群管理器管理，如Spark Standalone集群管理器、YARN集群管理器、Mesos
集群管理器。

spark应用程序

spark应用程序由一个driver进程（驱动程序）和一组Executor进程组成。

driver进行负责运行main函数，此进程位于集群中的一个节点上。负责三件事：维护有关spark应用程序的信息；响应
用户的应用程序或输入；分析、分配和调度executor。

executor进行只负责两件事：执行由驱动程序分配给它的代码；并将执行器executor的计算状态报告给驱动节点。

要点：

spark使用一个集群管理器来跟踪可用的资源

driver驱动程序负责执行驱动程序的命令，在executor执行器中完成给定的任务。

DataFrames

spark有几个核心抽象：Datasets、DataFrames、SQL表和弹性分布式数据集（RDDS）

Partitions

为了使每个executor执行器并行执行任务，Spark将数据分成块，这些数据块成为partition(分区)。
一个分区是集群中的一个物理机器上的行集合。

Transformations

在spark中核心数据结构是不可变的，这意味着它们在创建之后无法更改。要“更改”一个DataFrame，需要指导spark
如何修改它以实现想要的功能。这些指导指令称为Transformations转换。

Transformations转换是使用Spark表达业务逻辑的核心方法。
有两种类型的转换：窄依赖的转换、宽依赖的转换

窄依赖的转换是每个输入数据分区只对一个数据输出分区，一个输入分区最多一个输出分区。窄依赖转换，spark将自动
执行流水线pipeline的操作。

宽依赖的转换，一个输入数据分区对应多个输出分区。

Lazy Evaluation

延迟 evaulation 意味着 Spark将等到最后一刻才执行计算一些列指令。在spark中，不会执行某个转换操作时立刻修改数据，spark会构建一个你想要应用于源数据的转换计划。通过等待直到最后一分钟执行代码，spark将这个计划从原始的DataFrame转换到一个流线型的物理计划，该计划将在整个集群中尽可能高效的运行。这提供了巨大的好处，因为spark可以从端到端优化整个数据流。

Actions

Transformations转换允许我们构建逻辑转换计划。为了触发计算，我们运行一个action操作。action操作指示Spark通过执行一系列transformation转换计算结果。action 操作指示 Spark 通过执行一系列 transformation转换计算结果。

spark UI

通过spark web UI监视作业的进度。Spark UI在driver节点的4040端口上可用。如果在本地模式下，则将是HTTP://localhost:4040。Spark UI显示关于spark作业状态、环境和集群状态的信息。对于调优和调试非常有用。
